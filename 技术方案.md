# 新生报到系统 3.0 - 技术方案

## 一、项目概述

### 1.1 升级目标

在 2.0 版本"个人信息采集"的基础上，增加手机号验证码登录和身份证照片上传功能，构建完整的身份认证体系。系统将实现：
- 新生通过手机号验证身份
- 身份证正反面照片采集与存储
- 自动识别身份证正反面
- 防止重复照片上传

### 1.2 设计思想

采用分层架构设计，实现前后端分离，确保系统的可扩展性和可维护性。核心设计原则：
- **安全性优先**：验证码机制、权限控制、文件安全校验
- **用户体验优化**：自动识别、智能纠正、清晰的错误提示
- **技术选型合理性**：选择成熟稳定的技术栈

### 2.3 技术选型论证

#### 2.3.1 后端框架选择

| 选项 | 优势 | 劣势 | 决策 |
|------|------|------|------|
| Django | 成熟稳定、内置Session、ORM完善、媒体文件处理成熟 | 相对重量级 | ✅ 选择 |
| Flask | 轻量灵活 | 需要手动配置Session和文件上传 | ❌ 放弃 |
| FastAPI | 高性能、现代设计 | 生态相对较小 | ❌ 放弃 |

**选择理由**：Django提供了完整的Session管理机制和媒体文件处理能力，能够快速构建稳定的Web应用，符合本项目快速开发的需求。

#### 2.3.2 数据库选择

| 选项 | 优势 | 劣势 | 决策 |
|------|------|------|------|
| SQLite | 无需安装配置、文件型数据库、Django默认支持 | 并发性能有限 | ✅ 选择 |
| PostgreSQL | 性能优异、功能强大 | 需要独立部署维护 | ❌ 开发阶段暂不使用 |
| MySQL | 成熟稳定、社区活跃 | 需要独立部署维护 | ❌ 开发阶段暂不使用 |

**选择理由**：SQLite无需额外安装配置，非常适合开发阶段使用，能够快速启动项目。生产环境可无缝迁移至PostgreSQL。

#### 2.3.3 文件存储选择

| 选项 | 优势 | 劣势 | 决策 |
|------|------|------|------|
| 本地媒体目录 | 开发简单、访问快速 | 扩展性有限 | ✅ 开发阶段选择 |
| 云存储(OSS/COS) | 高可用、可扩展 | 配置复杂、有成本 | ❌ 生产环境考虑 |

**选择理由**：开发阶段使用本地存储简化配置，生产环境可迁移至云存储服务。

## 三、核心组件设计

### 3.1 身份认证组件

#### 3.1.1 验证码机制设计

**设计思路**：采用"手机号+验证码"的认证方式，验证码通过控制台打印（开发阶段），生产环境接入短信服务。

**核心流程**：
1. 用户输入手机号请求验证码
2. 后端生成4位随机数字验证码
3. 记录验证码到数据库，设置过期时间（5分钟）
4. 控制台打印验证码（开发阶段）
5. 用户输入验证码进行校验
6. 校验通过后建立Session

**安全设计**：
- 60秒发送频率限制
- 5分钟有效期
- 验证码使用后立即失效
- Session过期时间设置为2小时

### 3.2 文件上传组件

#### 3.2.1 身份证照片上传流程

```
前端选择图片 → 前端MD5检测 → 后端格式校验 → 后端重复检测 → 正反面识别 → 保存文件 → 记录路径
```

#### 3.2.2 安全校验机制

| 校验项 | 规则 | 实现方式 |
|--------|------|----------|
| 文件类型 | 仅允许.jpg/.png | 后端MIME类型检查 |
| 文件大小 | 单张≤5MB | 后端文件大小检查 |
| 重复检测 | MD5 + 感知哈希 | 双重检测算法 |
| 正反面识别 | 颜色空间分析 | OpenCV图像处理 |

### 3.3 身份证正反面识别组件

#### 3.3.1 识别算法设计

**设计思路**：基于图片颜色特征进行识别，身份证正面以蓝色为主且包含人脸区域，反面以红色为主（国徽）。

**算法流程**：
1. 将图片转换为HSV颜色空间
2. 检测蓝色区域比例（正面特征）
3. 检测肤色区域（人脸特征）
4. 检测红色区域比例（反面特征）
5. 综合判断正反面

**关键参数**：
- 蓝色阈值：Hue 100-130，Saturation 50-255，Value 50-255
- 肤色阈值：Hue 0-20，Saturation 20-255，Value 70-255
- 红色阈值：Hue 0-10，Saturation 100-255，Value 100-255

#### 3.3.2 识别逻辑

```python
def identify_id_card_side(image):
    """
    识别身份证正反面
    返回: 'front' | 'back' | 'unknown'
    """
    # 1. 颜色空间转换
    hsv = cv2.cvtColor(image, cv2.COLOR_BGR2HSV)
    
    # 2. 检测蓝色区域（正面特征）
    blue_mask = cv2.inRange(hsv, lower_blue, upper_blue)
    blue_ratio = np.sum(blue_mask > 0) / total_pixels
    
    # 3. 检测肤色区域（人脸特征）
    skin_mask = cv2.inRange(hsv, lower_skin, upper_skin)
    # 形态学操作去除噪声
    skin_mask = cv2.morphologyEx(skin_mask, cv2.MORPH_CLOSE, kernel)
    skin_mask = cv2.morphologyEx(skin_mask, cv2.MORPH_OPEN, kernel)
    # 检测连通区域判断是否有人脸
    contours, _ = cv2.findContours(skin_mask, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
    has_face = any(area > threshold for area in contour_areas)
    
    # 4. 检测红色区域（反面特征）
    red_mask = cv2.inRange(hsv, lower_red, upper_red)
    red_ratio = np.sum(red_mask > 0) / total_pixels
    
    # 5. 综合判断
    if blue_ratio > 0.02 and has_face:
        return 'front'
    elif red_ratio > 0.02 and not has_face:
        return 'back'
    # 兜底判断
    elif blue_ratio > 0.10:
        return 'front'
    else:
        return 'back'
```

### 5.2 接口详细设计

#### 5.2.1 发送验证码接口

**请求**：
```json
{
    "phone": "13800138000"
}
```

**响应**：
```json
{
    "success": true,
    "message": "验证码已发送"
}
```

**处理流程**：
1. 校验手机号格式（11位数字）
2. 检查60秒内是否已发送
3. 生成4位随机验证码
4. 计算过期时间（当前+5分钟）
5. 保存到数据库
6. 控制台打印验证码

#### 5.2.2 验证码校验接口

**请求**：
```json
{
    "phone": "13800138000",
    "code": "1234"
}
```

**响应**：
```json
{
    "success": true,
    "message": "登录成功",
    "data": {
        "phone": "13800138000"
    }
}
```

**处理流程**：
1. 查询最新未使用的验证码记录
2. 校验验证码是否匹配
3. 校验是否在有效期内
4. 标记验证码为已使用
5. 建立Session

#### 5.2.3 信息提交接口

**请求**：FormData
| 字段 | 类型 | 说明 |
|------|------|------|
| name | String | 姓名 |
| student_id | String | 学号 |
| id_card_photo_front | File | 身份证正面 |
| id_card_photo_back | File | 身份证反面 |

**响应**：
```json
{
    "success": true,
    "message": "报到成功",
    "data": {
        "name": "张三",
        "student_id": "20260001",
        "phone": "13800138000"
    }
}
```

**处理流程**：
1. 检查登录状态
2. 校验文本字段完整性
3. 校验图片格式和大小
4. MD5检测是否为同一张照片
5. 识别身份证正反面
6. 自动纠正上传顺序（如需要）
7. 保存图片到媒体目录
8. 创建学生信息记录
9. 创建报到记录
