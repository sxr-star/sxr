# 新生报到系统 4.0 - 技术方案

## 一、项目概述

### 1.1 升级目标

在 3.0 版本"手机号验证登录 + 信息采集 + 身份证上传"的基础上，新增管理员审核和报到数据统计看板两个功能模块。系统将实现：
- 新生提交报到信息后进入待审核状态
- 管理员审核学生信息（通过/驳回）
- 审核通过时分配宿舍号
- 新生可查看审核状态和宿舍信息
- 管理员通过统计看板实时查看报到数据

### 1.2 设计思想

采用分层架构设计，在保持3.0版本架构的基础上，扩展审核流程和数据分析能力。核心设计原则：
- **流程完整性**：构建"提交→审核→反馈"的完整闭环
- **权限隔离**：严格区分新生端和管理员端权限
- **数据实时性**：统计数据实时查询，确保数据准确性
- **可扩展性**：为后续5.0版本校园导览功能预留扩展空间

### 2.3 技术选型论证

#### 2.3.1 后端框架选择

| 选项 | 优势 | 劣势 | 决策 |
|------|------|------|------|
| Django | 成熟稳定、内置Session、ORM完善、媒体文件处理成熟、内置Admin后台 | 相对重量级 | ✅ 选择 |
| Flask | 轻量灵活 | 需要手动配置Session和文件上传 | ❌ 放弃 |
| FastAPI | 高性能、现代设计 | 生态相对较小 | ❌ 放弃 |

**选择理由**：Django提供了完整的Session管理机制、媒体文件处理能力和强大的Admin后台，能够快速构建稳定的Web应用和管理后台，符合本项目快速开发的需求。

#### 2.3.2 数据库选择

| 选项 | 优势 | 劣势 | 决策 |
|------|------|------|------|
| SQLite | 无需安装配置、文件型数据库、Django默认支持 | 并发性能有限 | ✅ 选择 |
| PostgreSQL | 性能优异、功能强大 | 需要独立部署维护 | ❌ 开发阶段暂不使用 |
| MySQL | 成熟稳定、社区活跃 | 需要独立部署维护 | ❌ 开发阶段暂不使用 |

**选择理由**：SQLite无需额外安装配置，非常适合开发阶段使用，能够快速启动项目。生产环境可无缝迁移至PostgreSQL。

#### 2.3.3 文件存储选择

| 选项 | 优势 | 劣势 | 决策 |
|------|------|------|------|
| 本地媒体目录 | 开发简单、访问快速 | 扩展性有限 | ✅ 开发阶段选择 |
| 云存储(OSS/COS) | 高可用、可扩展 | 配置复杂、有成本 | ❌ 生产环境考虑 |

**选择理由**：开发阶段使用本地存储简化配置，生产环境可迁移至云存储服务。

### 3.1 身份认证组件（保留3.0）

#### 3.1.1 验证码机制设计

**设计思路**：采用"手机号+验证码"的认证方式，验证码通过控制台打印（开发阶段），生产环境接入短信服务。

**核心流程**：
1. 用户输入手机号请求验证码
2. 后端生成4位随机数字验证码
3. 记录验证码到数据库，设置过期时间（5分钟）
4. 控制台打印验证码（开发阶段）
5. 用户输入验证码进行校验
6. 校验通过后建立Session

**安全设计**：
- 60秒发送频率限制
- 5分钟有效期
- 验证码使用后立即失效
- Session过期时间设置为2小时

### 3.3 身份证正反面识别组件（保留3.0）

#### 3.3.1 识别算法设计

**设计思路**：基于图片颜色特征进行识别，身份证正面以蓝色为主且包含人脸区域，反面以红色为主（国徽）。

**算法流程**：
1. 将图片转换为HSV颜色空间
2. 检测蓝色区域比例（正面特征）
3. 检测肤色区域（人脸特征）
4. 检测红色区域比例（反面特征）
5. 综合判断正反面

**关键参数**：
- 蓝色阈值：Hue 100-130，Saturation 50-255，Value 50-255
- 肤色阈值：Hue 0-20，Saturation 20-255，Value 70-255
- 红色阈值：Hue 0-10，Saturation 100-255，Value 100-255

#### 3.3.2 识别逻辑

```python
def identify_id_card_side(image):
    """
    识别身份证正反面
    返回: 'front' | 'back' | 'unknown'
    """
    # 1. 颜色空间转换
    hsv = cv2.cvtColor(image, cv2.COLOR_BGR2HSV)
    
    # 2. 检测蓝色区域（正面特征）
    blue_mask = cv2.inRange(hsv, lower_blue, upper_blue)
    blue_ratio = np.sum(blue_mask > 0) / total_pixels
    
    # 3. 检测肤色区域（人脸特征）
    skin_mask = cv2.inRange(hsv, lower_skin, upper_skin)
    # 形态学操作去除噪声
    skin_mask = cv2.morphologyEx(skin_mask, cv2.MORPH_CLOSE, kernel)
    skin_mask = cv2.morphologyEx(skin_mask, cv2.MORPH_OPEN, kernel)
    # 检测连通区域判断是否有人脸
    contours, _ = cv2.findContours(skin_mask, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
    has_face = any(area > threshold for area in contour_areas)
    
    # 4. 检测红色区域（反面特征）
    red_mask = cv2.inRange(hsv, lower_red, upper_red)
    red_ratio = np.sum(red_mask > 0) / total_pixels
    
    # 5. 综合判断
    if blue_ratio > 0.02 and has_face:
        return 'front'
    elif red_ratio > 0.02 and not has_face:
        return 'back'
    # 兜底判断
    elif blue_ratio > 0.10:
        return 'front'
    else:
        return 'back'
```

### 5.2 新增接口详细设计

#### 5.2.1 状态查询接口

**请求**：GET /api/review_status/

**响应**：
```json
{
    "success": true,
    "data": {
        "review_status": "pending",
        "reject_reason": null,
        "dormitory_number": null
    }
}
```

**处理流程**：
1. 检查登录状态（Session）
2. 根据Session中的手机号查询学生信息
3. 返回审核状态、驳回原因、宿舍号

#### 5.2.2 管理员审核接口

**请求**：POST /api/admin/review/

**请求体**：
```json
{
    "student_id": "20260001",
    "action": "approve",
    "dormitory_number": "A栋101",
    "reject_reason": null
}
```

**响应**：
```json
{
    "success": true,
    "message": "审核成功"
}
```

**处理流程**：
1. 校验管理员权限（is_staff）
2. 根据学号查询学生记录
3. 根据action执行审核操作：
   - approve：设置review_status='approved'，保存dormitory_number
   - reject：设置review_status='rejected'，保存reject_reason
4. 返回操作结果

**错误处理**：
- 非管理员：返回403
- 学号不存在：返回404
- 驳回时未填写原因：返回400

#### 5.2.3 统计数据接口

**请求**：GET /api/admin/statistics/

**响应**：
```json
{
    "success": true,
    "data": {
        "total": 100,
        "pending": 30,
        "approved": 60,
        "rejected": 10,
        "pass_rate": "60.00%"
    }
}
```

**处理流程**：
1. 校验管理员权限（is_staff）
2. 查询学生信息表统计各状态人数
3. 计算通过率（保留两位小数）
4. 返回统计数据

## 八、安全设计

### 8.1 接口安全

| 措施 | 说明 |
|------|------|
| 登录状态校验 | 信息提交接口要求登录 |

| 措施 | 说明 |
|------|------|

| 大小限制 | 单张≤5MB |
| 路径安全 | 使用Django内置存储，防止路径遍历 |
| 内容校验 | 检查文件魔数，防止恶意文件 |

### 8.3 数据安全

| 措施 | 说明 |
|------|------|
| 敏感数据保护 | 身份证照片存储在安全目录 |

| 数据备份 | 定期备份数据库和媒体文件 |
