## 2. 后端 API 测试

### 2.1 接口信息

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 首页 | GET | `/` | 返回网页版前端页面 |
| 报到接口 | POST | `/api/register/` | 接收报到请求，创建操作记录 |

### 2.2 测试用例

#### 测试用例 1：首页访问

| 项目 | 内容 |
|------|------|
| 测试步骤 | 浏览器访问 `http://127.0.0.1:8000/` |
| 预期结果 | 返回网页版前端页面，显示"欢迎使用新生报到系统"标题和"点击报到"按钮 |
| 实际结果 | ✅ 通过 |
| 备注 | 页面正常渲染，样式和布局正确 |

#### 测试用例 2：报到接口 - 正常请求

| 项目 | 内容 |
|------|------|
| 测试步骤 | 发送 POST 请求到 `/api/register/` |
| 预期结果 | 返回 `{"success": true, "message": "报到成功"}`，数据库中新增一条记录 |
| 实际结果 | ✅ 通过 |
| 备注 | 接口响应正常，数据正确写入数据库 |

#### 测试用例 3：报到接口 - GET 请求（应被拒绝）

| 项目 | 内容 |
|------|------|
| 测试步骤 | 发送 GET 请求到 `/api/register/` |
| 预期结果 | 返回 405 Method Not Allowed |
| 实际结果 | ✅ 通过 |
| 备注 | 接口使用了 `@require_POST` 装饰器，正确拒绝非 POST 请求 |

#### 测试用例 4：报到接口 - CSRF 豁免

| 项目 | 内容 |
|------|------|
| 测试步骤 | 发送不带 CSRF Token 的 POST 请求 |
| 预期结果 | 请求成功，返回报到成功 |
| 实际结果 | ✅ 通过 |
| 备注 | 接口使用了 `@csrf_exempt` 装饰器，允许跨域请求 |
