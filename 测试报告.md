# 新生报到系统 3.0 - 测试报告

## 1. 测试概述

### 1.1 测试目的

验证新生报到系统 3.0 版本（身份认证+身份证上传版）的各项功能是否正常运行，确保从手机号验证码登录到前端表单到后端 API 到数据库存储的完整数据链路畅通，学生信息采集、身份证照片上传与报到记录关联功能正确实现。

### 1.2 测试环境

| 项目 | 配置 |
|------|------|
| 操作系统 | Windows |
| Python 版本 | 3.10+ |
| Django 版本 | 4.2 |
| 数据库 | SQLite 3 |
| 图像处理 | OpenCV + NumPy |
| 浏览器 | Chrome / Edge |
| 测试日期 | 2026-05-19 |

### 1.3 测试范围

| 模块 | 说明 |
|------|------|
| 验证码发送接口 | 发送验证码、频率限制、控制台打印 |
| 验证码校验接口 | 验证码匹配、有效期校验、Session 登录 |
| 后端 API | 信息采集+身份证上传接口功能测试 |
| 网页版前端 | 验证页面、表单填写、图片上传与提交功能测试 |
| 小程序前端 | 验证页面、表单填写、图片上传与提交功能测试 |
| 数据库 | 验证码记录表、学生信息表与报到记录表关联测试 |
| 管理后台 | Django Admin 学生信息管理、身份证照片查看功能测试 |
| 数据校验 | 前端与后端双重校验测试 |
| 身份证识别 | 正反面自动识别、重复照片检测测试 |

## 2. 后端 API 测试

### 2.1 接口信息

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 首页 | GET | `/` | 返回网页版前端页面 |
| 发送验证码接口（3.0新增） | POST | `/api/send_code/` | 发送验证码到手机 |
| 验证码校验与登录接口（3.0新增） | POST | `/api/verify_code/` | 校验验证码并建立Session |
| 信息采集报到接口（2.0） | POST | `/api/register_with_info/` | 接收学生信息并创建报到记录 |
| 信息采集+身份证上传接口（3.0新增） | POST | `/api/register_with_info_v3/` | 接收学生信息和身份证照片并创建报到记录 |
