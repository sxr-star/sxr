## 1. 系统简介

新生报到系统 2.0 是一款面向高校新生报到场景的信息采集与管理平台。系统支持网页版和微信小程序两种访问方式，新生可通过填写姓名、学号、手机号完成报到操作，管理员可通过管理后台查看和管理报到数据。

### 1.1 核心功能

| 功能 | 说明 |
|------|------|
| 学生信息采集 | 收集新生姓名、学号、手机号 |
| 报到记录管理 | 自动记录报到时间，关联学生信息 |
| 数据验证 | 前端+后端双重验证，确保数据准确性 |
| 管理后台 | 查看、搜索、筛选报到数据 |
| 防重复报到 | 学号唯一性约束，防止重复提交 |

### 1.2 技术架构

| 层级 | 技术 |
|------|------|
| 后端 | Django 6.0.4 + Python 3.13.2 |
| 数据库 | SQLite 3 |
| 网页前端 | HTML5 + CSS3 + JavaScript |
| 小程序 | 微信小程序原生开发 |
| 版本控制 | Git + GitHub |

## 3. 安装部署

### 3.1 下载代码

```bash
git clone -b version2 https://github.com/sxr-star/sxr.git
cd sxr
```

### 3.2 安装依赖

**方式一：使用安装脚本（推荐）**

双击运行 `完整安装启动.bat`

**方式二：手动安装**

```bash
pip install django django-cors-headers
```

### 3.3 初始化数据库

```bash
python manage.py migrate
```

### 3.4 创建管理员账号

```bash
python manage.py createsuperuser
```

按提示输入：
- 用户名（Username）
- 邮箱（Email，可选）
- 密码（Password）
- 确认密码（Password again）

### 3.5 启动服务器

**方式一：使用启动脚本（推荐）**

双击运行 `快速启动.bat`

**方式二：命令行启动**

```bash
python manage.py runserver 8000
```

### 3.6 访问系统

| 入口 | 地址 |
|------|------|
| 网页版 | http://127.0.0.1:8000/ |
| 管理后台 | http://127.0.0.1:8000/admin/ |
| API接口 | http://127.0.0.1:8000/api/register_with_info/ |

## 4. 网页版使用指南

### 4.1 访问网页版

在浏览器地址栏输入：`http://127.0.0.1:8000/`

### 4.2 页面说明

| 区域 | 说明 |
|------|------|
| 标题栏 | 显示"欢迎使用新生报到系统" |
| 表单区 | 包含姓名、学号、手机号输入框 |
| 提交按钮 | 点击提交报到信息 |
| 结果区 | 显示报到成功/失败信息 |

### 4.3 操作步骤

**步骤 1：填写姓名**
- 在"姓名"输入框中输入真实姓名
- 必填项，不能为空

**步骤 2：填写学号**
- 在"学号"输入框中输入学校分配的学号
- 必填项，不能为空
- 学号唯一，不可重复报到

**步骤 3：填写手机号**
- 在"手机号"输入框中输入11位手机号码
- 必填项，不能为空
- 必须为11位数字

**步骤 4：提交报到**
- 确认信息填写无误后，点击"提交报到"按钮
- 按钮会显示"提交中..."，表示正在提交
- 提交成功后，页面会显示报到成功信息和报到时间

### 4.4 表单验证

| 验证项 | 规则 | 错误提示 |
|--------|------|---------|
| 姓名 | 不能为空 | "姓名不能为空" |
| 学号 | 不能为空 | "学号不能为空" |
| 手机号 | 不能为空 | "手机号不能为空" |
| 手机号 | 必须为11位数字 | "手机号格式不正确" |
| 学号 | 不可重复 | "学号已存在，请勿重复报到" |

### 4.5 注意事项

1. 请确保服务器已启动后再访问网页版
2. 填写信息时请仔细核对，提交后不可修改
3. 如遇到"网络请求失败"，请检查服务器是否正常运行
4. 每个学号只能报到一次，请勿重复提交

## 5. 小程序使用指南

### 5.1 配置小程序

**开发环境配置：**

1. 打开微信开发者工具
2. 导入小程序项目（`小程序` 目录）
3. 在"详情" -> "本地设置"中勾选"不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"
4. 确保后端服务器已启动

### 5.2 页面说明

| 区域 | 说明 |
|------|------|

| 提交按钮 | 点击提交报到信息 |
| 结果区 | 显示报到成功/失败信息 |

### 5.3 操作步骤

**步骤 1：填写姓名**
- 点击"姓名"输入框，输入真实姓名

**步骤 2：填写学号**
- 点击"学号"输入框，输入学校分配的学号

**步骤 3：填写手机号**
- 点击"手机号"输入框，输入11位手机号码

**步骤 4：提交报到**

## 6. 管理后台使用指南

### 6.1 登录管理后台

1. 在浏览器地址栏输入：`http://127.0.0.1:8000/admin/`
2. 输入管理员用户名和密码
3. 点击"登录"

### 6.2 功能模块

| 模块 | 说明 |
|------|------|
| 学生信息 | 查看所有已报到的学生信息 |
| 报到记录 | 查看所有报到记录及关联信息 |
| 用户和组 | 管理管理员账号和权限 |

### 6.3 学生信息管理

**查看列表：**
- 点击"学生信息"进入列表页
- 显示字段：ID、姓名、学号、手机号、创建时间
- 默认按创建时间倒序排列

**搜索功能：**
- 在右上角搜索框输入学号或姓名
- 点击"搜索"按钮进行筛选

**筛选功能：**
- 使用右侧"创建时间"筛选器
- 可按日期范围筛选学生

**查看详情：**
- 点击任意学生记录进入详情页
- 可查看完整的学生信息

### 6.4 报到记录管理

**查看列表：**
- 点击"报到记录"进入列表页
- 显示字段：ID、姓名、学号、报到时间
- 默认按报到时间倒序排列

**搜索功能：**
- 在右上角搜索框输入学号或姓名
- 点击"搜索"按钮进行筛选

**查看详情：**
- 点击任意报到记录进入详情页
- 可查看关联的学生信息

### 6.5 注意事项

1. 管理员账号需通过 `createsuperuser` 命令创建
2. 请勿泄露管理员账号和密码
3. 定期备份数据库文件（`db.sqlite3`）
4. 生产环境建议修改默认数据库为 MySQL 或 PostgreSQL

## 7. 常见问题

### 7.1 安装部署问题

**Q1：运行 `python manage.py runserver` 报错 "No module named django"**

A：未安装Django，请运行以下命令安装：
```bash
pip install django django-cors-headers
```

**Q2：运行 `python manage.py migrate` 报错**

A：请确保当前目录为项目根目录（包含 `manage.py` 文件的目录）

**Q3：服务器启动后无法访问**

A：请检查：
1. 端口8000是否被占用
2. 防火墙是否阻止访问
3. 服务器是否成功启动（查看终端输出）

**Q4：如何更换端口？**

A：使用以下命令指定端口：
```bash
python manage.py runserver 8080
```

### 7.2 网页版问题

**Q5：访问网页版显示404**

A：请确保访问的是根路径 `/`，而不是其他路径

**Q6：点击"提交报到"按钮后显示"网络请求失败"**

A：请检查：
1. Django服务器是否正在运行
2. 浏览器控制台是否有错误信息
3. 网络连接是否正常

**Q7：提交后显示"学号已存在"**

A：该学号已经报到过，请勿重复提交

### 7.3 小程序问题

**Q8：小程序请求报错 "request:fail url not in domain list"**

A：请在微信开发者工具中勾选"不校验合法域名"

**Q9：小程序无法连接服务器**

A：请确保：
1. 后端服务器已启动
2. 小程序中的服务器地址正确
3. 手机和电脑在同一网络环境下（真机调试时）

### 7.4 管理后台问题

**Q10：忘记管理员密码**

A：可通过以下命令重置：
```bash
python manage.py changepassword 用户名
```

**Q11：如何创建新的管理员账号？**

A：运行以下命令：
```bash
python manage.py createsuperuser
```

### 7.5 数据库问题

**Q12：如何备份数据库？**

A：复制 `新生报到系统/db.sqlite3` 文件到其他位置

**Q13：如何恢复数据库？**

A：将备份的 `db.sqlite3` 文件复制回 `新生报到系统/` 目录

**Q14：如何导出报到数据？**

A：可通过管理后台查看，或使用以下命令导出：
```bash
python manage.py dumpdata api > backup.json
```
