## 1. 系统简介

新生报到系统 3.0 是一款面向高校新生报到场景的信息采集与管理平台。系统在 2.0 版本基础上增加了手机号验证码登录和身份证照片上传功能。新生需先通过手机号验证身份，然后填写个人信息并上传身份证正反面照片完成报到。系统支持网页版和微信小程序两种访问方式。

### 1.1 核心功能

| 功能 | 说明 |
|------|------|
| 手机号验证码登录 | 发送验证码到手机，验证后登录系统 |
| 学生信息采集 | 收集新生姓名、学号、手机号 |
| 身份证照片上传 | 上传身份证正反面照片，自动识别正反面 |
| 重复照片检测 | 检测是否上传同一张照片（MD5 + 感知哈希） |
| 报到记录管理 | 自动记录报到时间，关联学生信息 |
| 数据验证 | 前端+后端双重验证，确保数据准确性 |
| 管理后台 | 查看、搜索、筛选报到数据，查看身份证照片 |
| 防重复报到 | 学号唯一性约束，防止重复提交 |

### 1.2 技术架构

| 层级 | 技术 |
|------|------|
| 后端 | Django 4.2 + Python 3.10+ |
| 数据库 | SQLite 3 |
| 图像处理 | OpenCV + NumPy |
| 网页前端 | HTML5 + CSS3 + JavaScript + CryptoJS |
| 小程序 | 微信小程序原生开发 |
| 版本控制 | Git + GitHub |

## 3. 安装部署

### 3.1 下载代码

```bash
git clone -b version2 https://github.com/sxr-star/sxr.git
cd sxr
```

### 3.2 安装依赖

**方式一：使用安装脚本（推荐）**

双击运行 `完整安装启动.bat`

**方式二：手动安装**

```bash
pip install django django-cors-headers opencv-python numpy pillow
```

### 3.3 初始化数据库

```bash
python manage.py migrate
```

### 3.4 创建管理员账号

```bash
python manage.py createsuperuser
```

按提示输入：
- 用户名（Username）
- 邮箱（Email，可选）
- 密码（Password）
- 确认密码（Password again）

### 3.5 启动服务器

**方式一：使用启动脚本（推荐）**

双击运行 `快速启动.bat`

**方式二：命令行启动**

```bash
python manage.py runserver 8000
```

### 3.6 访问系统

| 入口 | 地址 |
|------|------|
| 网页版 | http://127.0.0.1:8000/ |
| 管理后台 | http://127.0.0.1:8000/admin/ |
| API接口 | http://127.0.0.1:8000/api/ |

## 4. 网页版使用指南

### 4.1 访问网页版

在浏览器地址栏输入：`http://127.0.0.1:8000/`

### 4.2 页面说明

系统包含两个页面：验证页面和报到页面。

#### 4.2.1 验证页面

| 区域 | 说明 |
|------|------|
| 标题栏 | 显示"新生报到系统 - 身份验证" |
| 手机号输入框 | 输入11位手机号码 |
| 获取验证码按钮 | 点击发送验证码，60秒倒计时 |
| 验证码输入框 | 输入收到的4位数字验证码 |
| 登录按钮 | 点击验证并登录 |

#### 4.2.2 报到页面

| 区域 | 说明 |
|------|------|
| 标题栏 | 显示"欢迎使用新生报到系统" |
| 手机号显示区 | 自动填充当前登录用户的手机号，不可编辑 |
| 表单区 | 包含姓名、学号输入框 |
| 身份证正面照片区 | 上传身份证正面（人像面）照片 |
| 身份证反面照片区 | 上传身份证反面（国徽面）照片 |
| 提交按钮 | 点击提交报到信息 |
| 结果区 | 显示报到成功/失败信息 |

### 4.3 操作步骤

#### 步骤一：手机号验证登录

**步骤 1：输入手机号**
- 在"手机号"输入框中输入11位手机号码
- 必须为有效的中国大陆手机号

**步骤 2：获取验证码**
- 点击"获取验证码"按钮
- 系统会发送4位数字验证码到该手机号
- 开发阶段验证码在服务器控制台打印
- 按钮进入60秒倒计时，期间不可重复点击

**步骤 3：输入验证码**
- 在"验证码"输入框中输入收到的4位数字验证码
- 验证码有效期为5分钟

**步骤 4：登录**
- 点击"登录"按钮
- 验证成功后自动跳转到报到页面

#### 步骤二：填写报到信息

**步骤 5：填写姓名**
- 在"姓名"输入框中输入真实姓名
- 必填项，不能为空

**步骤 6：填写学号**
- 在"学号"输入框中输入学校分配的学号
- 必填项，不能为空
- 学号唯一，不可重复报到

**步骤 7：上传身份证正面照片**
- 点击"选择正面照片"按钮
- 选择身份证正面（带人像的一面）照片
- 支持 .jpg、.png 格式，大小不超过 5MB
- 系统会自动识别是否为正面照片

**步骤 8：上传身份证反面照片**
- 点击"选择反面照片"按钮
- 选择身份证反面（带国徽的一面）照片
- 支持 .jpg、.png 格式，大小不超过 5MB
- 系统会自动识别是否为反面照片

**步骤 9：提交报到**
- 确认信息填写无误后，点击"提交报到"按钮
- 按钮会显示"提交中..."，表示正在提交
- 系统会自动检测：
  - 两张照片是否为同一张
  - 正反面是否识别正确
  - 如上传顺序错误会自动纠正
- 提交成功后，页面会显示报到成功信息和报到时间

### 4.4 表单验证

| 验证项 | 规则 | 错误提示 |
|--------|------|---------|
| 手机号 | 不能为空 | "手机号不能为空" |
| 手机号 | 必须为11位数字 | "手机号格式不正确" |
| 验证码 | 不能为空 | "请输入验证码" |
| 验证码 | 必须为4位数字 | "验证码格式不正确" |
| 验证码 | 60秒内不可重复发送 | "请等待XX秒后重试" |
| 姓名 | 不能为空 | "姓名不能为空" |
| 学号 | 不能为空 | "学号不能为空" |
| 身份证正面照片 | 必须上传 | "请上传身份证正面照片" |
| 身份证反面照片 | 必须上传 | "请上传身份证反面照片" |
| 照片格式 | .jpg 或 .png | "请上传jpg或png格式的图片" |
| 照片大小 | 不超过5MB | "图片大小不能超过5MB" |
| 照片重复 | 正反面不能为同一张 | "身份证正面和反面不能上传同一张图片" |
| 学号 | 不可重复 | "学号已存在，请勿重复报到" |

### 4.5 注意事项

1. 请确保服务器已启动后再访问网页版
2. 填写信息时请仔细核对，提交后不可修改
3. 如遇到"网络请求失败"，请检查服务器是否正常运行
4. 每个学号只能报到一次，请勿重复提交
5. 身份证照片请确保清晰可辨，系统会自动识别正反面
6. 如上传顺序错误，系统会自动纠正
7. 请勿上传同一张照片作为正反面

## 6. 管理后台使用指南

### 6.1 登录管理后台

1. 在浏览器地址栏输入：`http://127.0.0.1:8000/admin/`
2. 输入管理员用户名和密码
3. 点击"登录"

### 6.2 功能模块

| 模块 | 说明 |
|------|------|
| 学生信息 | 查看所有已报到的学生信息 |
| 报到记录 | 查看所有报到记录及关联信息 |
| 用户和组 | 管理管理员账号和权限 |

### 6.3 学生信息管理

**查看列表：**
- 点击"学生信息"进入列表页
- 显示字段：ID、姓名、学号、手机号、已上传正面照片、已上传反面照片、创建时间
- 默认按创建时间倒序排列

**搜索功能：**
- 在右上角搜索框输入学号、姓名或手机号
- 点击"搜索"按钮进行筛选

**筛选功能：**
- 使用右侧"创建时间"筛选器
- 可按日期范围筛选学生

**查看详情：**
- 点击任意学生记录进入详情页
- 可查看完整的学生信息
- 可查看上传的身份证正面和反面照片

### 6.4 报到记录管理

**查看列表：**
- 点击"报到记录"进入列表页
- 显示字段：ID、姓名、学号、报到时间
- 默认按报到时间倒序排列

**搜索功能：**
- 在右上角搜索框输入学号或姓名
- 点击"搜索"按钮进行筛选

**查看详情：**
- 点击任意报到记录进入详情页
- 可查看关联的学生信息

### 6.5 注意事项

1. 管理员账号需通过 `createsuperuser` 命令创建
2. 请勿泄露管理员账号和密码
3. 定期备份数据库文件（`db.sqlite3`）
4. 生产环境建议修改默认数据库为 MySQL 或 PostgreSQL

## 7. 常见问题

### 7.1 安装部署问题

**Q1：运行 `python manage.py runserver` 报错 "No module named django"**

A：未安装Django，请运行以下命令安装：
```bash
pip install django django-cors-headers opencv-python numpy pillow
```

**Q2：运行 `python manage.py migrate` 报错**

A：请确保当前目录为项目根目录（包含 `manage.py` 文件的目录）

**Q3：服务器启动后无法访问**

A：请检查：
1. 端口8000是否被占用
2. 防火墙是否阻止访问
3. 服务器是否成功启动（查看终端输出）

**Q4：如何更换端口？**

A：使用以下命令指定端口：
```bash
python manage.py runserver 8080
```

### 7.2 网页版问题

**Q5：访问网页版显示404**

A：请确保访问的是根路径 `/`，而不是其他路径

**Q6：点击"提交报到"按钮后显示"网络请求失败"**

A：请检查：
1. Django服务器是否正在运行
2. 浏览器控制台是否有错误信息
3. 网络连接是否正常

**Q7：提交后显示"学号已存在"**

A：该学号已经报到过，请勿重复提交

**Q8：提示"未检测到身份证正面照片"**

A：请检查上传的照片是否为身份证正面（带人像的一面），确保照片清晰可辨

**Q9：提示"未检测到身份证反面照片"**

A：请检查上传的照片是否为身份证反面（带国徽的一面），确保照片清晰可辨

**Q10：提示"身份证正面和反面不能上传同一张图片"**

A：请确保正面和反面分别上传了不同的照片

### 7.3 小程序问题

**Q11：小程序请求报错 "request:fail url not in domain list"**

A：请在微信开发者工具中勾选"不校验合法域名"

**Q12：小程序无法连接服务器**

A：请确保：
1. 后端服务器已启动
2. 小程序中的服务器地址正确
3. 手机和电脑在同一网络环境下（真机调试时）

### 7.4 管理后台问题

**Q13：忘记管理员密码**

A：可通过以下命令重置：
```bash
python manage.py changepassword 用户名
```

**Q14：如何创建新的管理员账号？**

A：运行以下命令：
```bash
python manage.py createsuperuser
```

**Q15：如何查看学生上传的身份证照片？**

A：进入学生信息详情页，点击照片链接即可查看

### 7.5 数据库问题

**Q16：如何备份数据库？**

A：复制 `新生报到系统/db.sqlite3` 文件到其他位置

**Q17：如何恢复数据库？**

A：将备份的 `db.sqlite3` 文件复制回 `新生报到系统/` 目录

**Q18：如何导出报到数据？**

A：可通过管理后台查看，或使用以下命令导出：
```bash
python manage.py dumpdata api > backup.json
```
