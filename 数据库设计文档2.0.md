# 新生报到系统 2.0 - 数据库设计文档

## 1. 文档概述

本文档描述新生报到系统 2.0 版本的数据库设计方案，详细说明数据库表结构、字段定义、索引设计以及 ER 图关系。

### 1.1 设计目标

- 实现新生个人信息采集与报到操作的数据持久化存储
- 支持学生信息表与报到记录表的一对一关联
- 支持管理后台对学生信息的查询、搜索和排序
- 保证学号唯一性，防止重复报到

### 1.2 设计原则

- 遵循数据库设计三大范式
- 字段命名规范统一
- 充分考虑查询性能
- 采用表分离设计，职责分明
- 为未来扩展预留空间

### 1.3 版本说明

| 版本 | 日期 | 说明 |
|------|------|------|
| 1.0 | 2026-04-24 | 初始版本，仅包含操作记录表 |
| 2.0 | 2026-04-27 | 新增学生信息表，实现一对一关联 |

## 3. 数据表设计

### 3.1 学生信息表（api_studentinfo）

#### 3.1.1 表结构说明

表名：`api_studentinfo`

用途：存储新生个人基本信息，包括姓名、学号、手机号等字段。

### 3.2 报到记录表（api_registrationrecord）

#### 3.2.1 表结构说明

表名：`api_registrationrecord`

用途：存储新生的报到记录，与学生信息表建立一对一关联，确保每个报到动作都对应一份完整的个人资料。

#### 3.2.2 字段详细说明

| 字段名 | 数据类型 | 是否可为空 | 默认值 | 主键 | 外键 | 说明 |
|--------|----------|------------|--------|------|------|------|
| id | IntegerField | NO | 自动递增 | ✅ | - | 自增主键，唯一标识每条报到记录 |
| student | OneToOneField | NO | - | - | api_studentinfo.id | 关联的学生信息 |
| register_time | DateTimeField | NO | 自动记录 | - | - | 报到时间 |

#### 3.2.3 字段详细定义

**id 字段**
```python
id = models.AutoField(primary_key=True)
```
- 数据类型：整数型
- 主键：是
- 自增：是
- 说明：系统自动生成的唯一标识符

**student 字段**
```python
student = models.OneToOneField(StudentInfo, on_delete=models.CASCADE, verbose_name='学生信息')
```
- 数据类型：外键（一对一关联）
- 关联表：api_studentinfo
- 删除策略：CASCADE（级联删除）
- 是否必填：是
- 中文标签：学生信息
- 说明：关联到学生信息表，确保每条报到记录对应一个学生

**register_time 字段**
```python
register_time = models.DateTimeField(auto_now_add=True, verbose_name='报到时间')
```
- 数据类型：日期时间型
- 自动填充：是（创建时自动记录）
- 中文标签：报到时间
- 说明：在记录生成时自动填入当前时间，精确到毫秒

#### 3.2.4 索引设计

| 索引名 | 索引类型 | 索引字段 | 说明 |
|--------|----------|----------|------|
| PRIMARY | 主键索引 | id | 自动创建，保证唯一性 |
| api_registrationrecord_student_id | 唯一索引 | student_id | 一对一关联，保证唯一 |
| api_registrationrecord_register_time | 普通索引 | register_time | 用于按时间排序查询 |

#### 3.2.5 表约束

| 约束类型 | 约束说明 |
|----------|----------|
| 主键约束 | id 字段为主键，确保唯一性 |
| 非空约束 | student、register_time 字段非空 |
| 外键约束 | student 字段关联 api_studentinfo.id |
| 唯一约束 | student 字段唯一，一个学生只能有一条报到记录 |
| 级联删除 | 删除学生信息时，自动删除关联的报到记录 |

## 5. SQL 语句

### 5.1 建表语句

```sql
-- 学生信息表
CREATE TABLE api_studentinfo (
    id INTEGER NOT NULL PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(50) NOT NULL,
    student_id VARCHAR(20) NOT NULL UNIQUE,
    phone VARCHAR(11) NOT NULL,
    created_at DATETIME NOT NULL
);

CREATE INDEX api_studentinfo_created_at ON api_studentinfo (created_at);

-- 报到记录表
CREATE TABLE api_registrationrecord (
    id INTEGER NOT NULL PRIMARY KEY AUTOINCREMENT,
    student_id INTEGER NOT NULL UNIQUE REFERENCES api_studentinfo(id),
    register_time DATETIME NOT NULL
);

CREATE INDEX api_registrationrecord_register_time ON api_registrationrecord (register_time);
```

### 5.2 插入数据

```sql
-- 插入学生信息
INSERT INTO api_studentinfo (name, student_id, phone, created_at) 
VALUES ('张三', '2024001001', '13800138000', '2024-09-01 10:30:00');

-- 插入报到记录（关联学生信息）
INSERT INTO api_registrationrecord (student_id, register_time) 
VALUES (1, '2024-09-01 10:30:00');
```

### 5.3 查询数据

```sql
-- 查询所有学生信息（按创建时间倒序）
SELECT * FROM api_studentinfo ORDER BY created_at DESC;

-- 查询所有报到记录及学生信息（关联查询）
SELECT 
    r.id,
    s.name,
    s.student_id,
    s.phone,
    r.register_time
FROM api_registrationrecord r
JOIN api_studentinfo s ON r.student_id = s.id
ORDER BY r.register_time DESC;

-- 按学号搜索学生
SELECT * FROM api_studentinfo WHERE student_id = '2024001001';

-- 统计报到总人数
SELECT COUNT(*) FROM api_registrationrecord;

-- 查询某学生的报到记录
SELECT 
    s.name,
    s.student_id,
    s.phone,
    r.register_time
FROM api_studentinfo s
JOIN api_registrationrecord r ON s.id = r.student_id
WHERE s.student_id = '2024001001';
```

### 5.4 更新数据

```sql
-- 更新学生手机号
UPDATE api_studentinfo SET phone = '13900139000' WHERE student_id = '2024001001';

-- 更新学生姓名
UPDATE api_studentinfo SET name = '李四' WHERE student_id = '2024001001';
```

### 5.5 删除数据

```sql
-- 删除学生信息（级联删除报到记录）
DELETE FROM api_studentinfo WHERE student_id = '2024001001';

-- 清空所有报到记录
DELETE FROM api_registrationrecord;

-- 清空所有学生信息
DELETE FROM api_studentinfo;
```

## 6. Django 模型定义

### 6.1 模型代码

```python
from django.db import models


class StudentInfo(models.Model):
    """学生信息表 - 存储新生个人基本信息"""
    
    name = models.CharField(
        max_length=50,
        verbose_name='姓名'
    )
    student_id = models.CharField(
        max_length=20,
        unique=True,
        verbose_name='学号'
    )
    phone = models.CharField(
        max_length=11,
        verbose_name='手机号'
    )
    created_at = models.DateTimeField(
        auto_now_add=True,
        verbose_name='创建时间'
    )

    class Meta:
        verbose_name = '学生信息'
        verbose_name_plural = '学生信息'
        ordering = ['-created_at']

    def __str__(self):
        return f"{self.name} - {self.student_id}"


class RegistrationRecord(models.Model):
    """报到记录表 - 存储新生报到记录，与学生信息一对一关联"""
    
    student = models.OneToOneField(
        StudentInfo,
        on_delete=models.CASCADE,
        verbose_name='学生信息'
    )
    register_time = models.DateTimeField(
        auto_now_add=True,
        verbose_name='报到时间'
    )

    class Meta:
        verbose_name = '报到记录'
        verbose_name_plural = '报到记录'
        ordering = ['-register_time']

    def __str__(self):
        return f"{self.student.name} - {self.register_time}"
```

### 6.2 模型说明

| 属性 | 说明 |
|------|------|
| verbose_name | 单数形式的中文名称 |
| verbose_name_plural | 复数形式的中文名称 |
| ordering | 默认排序方式（按时间倒序） |
| __str__ | 对象的字符串表示 |
| on_delete | 删除策略（CASCADE 级联删除） |

## 8. 接口设计

### 8.1 提交个人信息 + 报到接口

**请求路径**：`/api/register_with_info/`

**请求方法**：POST

**请求参数**（JSON 格式）：

```json
{
    "name": "张三",
    "student_id": "2024001001",
    "phone": "13800138000"
```


```json

    "message": "学号已存在，请勿重复报到"
```

```python
    student_id=student_id,
    phone=phone
```

## 12. 性能优化建议

### 12.1 索引优化

- 学号字段已设置唯一索引，加速查询和重复检查
- 时间字段已添加索引，支持后台快速排序
- 外键字段自动创建索引，加速关联查询

### 12.2 查询优化

- 使用 `select_related` 进行一对一关联查询，减少数据库查询次数
- 避免使用 `SELECT *`，只查询需要的字段
- 合理使用分页，避免一次性加载大量数据

### 12.3 示例优化查询

```python
# 优化前（N+1 查询问题）
records = RegistrationRecord.objects.all()
for record in records:
    print(record.student.name)

# 优化后（使用 select_related）
records = RegistrationRecord.objects.select_related('student').all()
for record in records:
    print(record.student.name)
```
