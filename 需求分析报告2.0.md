# 新生报到系统 2.0 - 需求报告

## 一、升级目标

在 1.0 版本"点击报到"极简链路的基础上，增加新生信息采集功能。让小程序和网页端不仅能报到，还能让新生填写个人基本资料，并将这些信息持久化存储，管理员可在后台统一查看。

**一句话目标**：新生报到时，先填表，再提交，报到记录中包含姓名、学号等信息。

---

## 二、新增功能范围

### ✅ 新增功能

| 功能 | 描述 |
|------|------|
| 个人信息填写表单 | 姓名、学号、手机号 |
| 表单数据持久化 | 数据存入数据库，与报到记录关联 |
| 多端同步更新 | 网页版和小程序端同步支持填表后报到 |

### ❌ 暂不加入功能

- 微信登录
- 图片上传
- 管理员审核
- 宿舍分配

---

## 三、功能需求详细说明

### 3.1 个人信息填写表单

#### 3.1.1 表单字段

| 字段名称 | 类型 | 必填 | 说明 |
|----------|------|------|------|
| 姓名 | 字符串 | 是 | 学生真实姓名 |
| 学号 | 字符串 | 是 | 学生学号，唯一标识 |
| 手机号 | 字符串 | 是 | 学生联系电话 |

#### 3.1.2 表单交互要求

- 表单包含三个输入框和一个"提交报到"按钮
- 前端需做非空校验
- 缺少任一字段点击提交时，前端给出提示，不发送请求

### 3.2 数据库模型升级

#### 3.2.1 学生信息表

扩展或新建学生信息表，包含以下字段：

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| 姓名 | 字符串 | 必填 | 学生姓名 |
| 学号 | 字符串 | 必填、唯一 | 学生学号 |
| 手机号 | 字符串 | 必填 | 联系电话 |

#### 3.2.2 关联关系

- 报到记录表与学生信息表建立一对一关联
- 确保每个报到动作都对应一份完整的个人资料

### 3.3 后端接口

#### 3.3.1 新接口定义

| 属性 | 值 |
|------|------|
| 接口名称 | 提交个人信息 + 报到 |
| 请求方式 | POST |
| 请求数据 | 姓名、学号、手机号 |

#### 3.3.2 接口逻辑

1. 接收前端发送的表单数据
2. 验证字段是否完整
3. 将数据写入学生信息表
4. 在报到记录表中生成一条关联记录
5. 返回操作结果（成功/失败及错误提示）

### 3.4 前端页面改动

#### 3.4.1 网页版

- 移除原有单一"点击报到"按钮
- 添加表单页面：三个输入框 + 一个提交按钮
- 使用 JavaScript 监听提交事件
- 通过 fetch 发送 POST 请求
- 根据响应展示成功/失败信息

#### 3.4.2 小程序版

- 将原有按钮替换为表单输入组件（input）
- 实现表单数据双向绑定
- 为"提交报到"按钮绑定事件
- 调用 wx.request 发送表单数据
- 处理回调，展示成功提示并清空表单

### 3.5 管理后台优化

- 将学生信息表注册到 Django Admin
- 管理员可按列表查看所有报到新生的姓名、学号、手机号及报到时间
- 支持按学号搜索
- 支持按时间排序

## 四、技术实现方案

### 4.1 数据库模型扩展

- 在 `api/models.py` 中新建学生信息模型
- 定义姓名、学号、手机号字段
- 与已有报到记录模型做一对一关联
- 执行迁移命令使新模型生效
- 在 `api/admin.py` 中注册新模型

### 4.2 后端视图修改

- 在 `api/views.py` 中编写新视图函数
- 函数逻辑：校验字段 → 创建学生信息记录 → 创建报到记录并关联 → 返回 JSON 响应
- 在 `api/urls.py` 中添加新接口路由

### 4.3 前端更新

- 网页版：修改 `网页版/index.html`，添加表单 HTML 和 JavaScript 逻辑
- 小程序版：修改小程序页面文件，添加表单组件和提交逻辑

## 六、验收标准

| 编号 | 验收项 | 标准 |
|------|--------|------|
| 1 | 表单展示 | 网页端和小程序端均能显示表单页面 |
| 2 | 前端校验 | 缺少任一字段点击提交时，前端给出提示，不发送请求 |
| 3 | 提交成功 | 正确填写三个字段后提交，收到成功响应 |
| 4 | 后台记录 | 后台管理界面中，学生信息表有对应记录，报到记录表有一条关联数据 |
| 5 | 唯一性约束 | 重复提交时，学号唯一性可被约束（数据库层面或后端校验） |

---

## 七、注意事项

1. 修改模型后务必做好数据备份（当前 SQLite 文件可复制一份）
2. 小程序开发期间保留"不校验合法域名"选项，避免本地调试被拦截
3. 前端请求地址需根据实际启动端口调整，保持一致
