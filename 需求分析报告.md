# 新生报到系统 - 需求分析报告

## 1. 项目背景

### 1.1 项目概述
新生报到系统是一个面向高校新生报到场景的信息化管理工具。该系统旨在简化新生报到流程，实现报到数据的电子化存储和管理，提高报到效率和管理水平。

### 1.2 项目目标
- 建立从前端到后端的完整数据通道
- 实现新生报到功能的在线化
- 提供管理后台查看报到记录
- 支持多端访问（网页版、微信小程序）

### 1.3 适用范围
本系统适用于高校新生报到场景，可作为学校信息化建设的组成部分。

---

## 2. 需求概述

### 2.1 核心需求
| 需求类型 | 描述 | 优先级 |
|---------|------|--------|
| 前端报到 | 用户通过网页或小程序点击按钮完成报到 | 高 |
| 后端接口 | 接收前端请求并存储到数据库 | 高 |
| 数据存储 | 使用 SQLite 数据库存储报到记录 | 高 |
| 管理后台 | 管理员可查看报到记录 | 高 |

### 2.2 非核心需求（本期不实现）
- 微信登录功能
- 复杂表单填写
- 图片上传功能
- 审批工作流
- 数据导出功能

---

## 3. 功能需求

### 3.1 用户端功能

#### 3.1.1 网页版报到
- **功能描述**：用户通过浏览器访问网页，点击报到按钮完成报到
- **操作流程**：
  1. 打开浏览器访问系统网址
  2. 点击"点击报到"按钮
  3. 系统显示报到结果（成功/失败）
- **技术要求**：
  - 使用 HTML/CSS/JavaScript 开发
  - 使用 fetch API 发送 POST 请求
  - 支持响应式布局

#### 3.1.2 微信小程序版
- **功能描述**：用户通过微信小程序点击报到按钮完成报到
- **操作流程**：
  1. 打开微信小程序
  2. 点击"点击报到"按钮
  3. 系统显示报到结果（成功/失败）
- **技术要求**：
  - 使用微信小程序开发框架
  - 使用 wx.request 发送 POST 请求
  - 支持加载状态显示

### 3.2 管理端功能

#### 3.2.1 管理后台
- **功能描述**：管理员通过 Django Admin 查看报到记录
- **操作流程**：
  1. 访问管理后台网址
  2. 输入管理员账号密码登录
  3. 查看报到记录列表
- **技术要求**：
  - 使用 Django Admin 框架
  - 支持记录查看、筛选、排序

## 4. 非功能需求

### 4.1 性能需求
- 页面加载时间 < 3 秒
- 接口响应时间 < 1 秒
- 支持并发用户数 ≥ 100

### 4.2 安全需求
- 管理后台需要账号密码认证
- API 接口启用 CSRF 保护
- 数据库文件定期备份

### 4.3 可用性需求
- 系统可用性 ≥ 99%
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 支持移动端访问

### 4.4 可维护性需求
- 代码结构清晰，注释完整
- 使用 Git 进行版本控制
- 提供完整的开发文档

## 6. 接口设计

### 6.1 报到接口

#### 请求信息
- **URL**：`/api/register/`
- **方法**：POST
- **Content-Type**：application/json

#### 请求参数
无（空请求体）

#### 响应参数
| 字段名 | 类型 | 描述 |
|--------|------|------|
| success | Boolean | 是否成功 |
| message | String | 提示信息 |

#### 响应示例
```json
{
    "success": true,
    "message": "报到成功"
}
```

## 7. 部署方案

### 7.1 开发环境
- **操作系统**：Windows 10/11
- **Python 版本**：3.14+
- **Django 版本**：6.0.4
- **数据库**：SQLite（内置）

### 7.2 部署步骤
1. 安装 Python
2. 安装依赖：`pip install django django-cors-headers`
3. 执行数据库迁移：`python manage.py migrate`
4. 创建管理员账号：`python manage.py createsuperuser`
5. 启动服务器：`python manage.py runserver 8000`

### 7.3 访问地址
| 页面 | 地址 |
|------|------|
| 网页版报到页面 | http://127.0.0.1:8000/ |
| 管理后台 | http://127.0.0.1:8000/admin |
| API 接口 | http://127.0.0.1:8000/api/register/ |

## 10. 风险评估

### 10.1 技术风险
| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 网络问题导致推送失败 | 代码无法同步 | 使用一键推送脚本 |
| 跨域问题 | 前端无法请求后端 | 配置 CORS 中间件 |
| 数据库迁移失败 | 表不存在 | 重新执行 migrate |

### 10.2 业务风险
| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 并发量大 | 系统响应慢 | 优化数据库查询 |
| 数据丢失 | 报到记录丢失 | 定期备份数据库 |
