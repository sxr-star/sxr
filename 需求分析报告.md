# 新生报到系统 3.0 - 需求分析报告

## 一、引言

### 1.1 项目背景

随着高校信息化建设的推进，传统的新生报到流程已难以满足高效、便捷的服务需求。本系统旨在通过数字化手段，实现新生报到流程的线上化、智能化，提升报到效率和服务质量。

### 1.2 文档目的

本需求分析报告详细描述新生报到系统 3.0 版本的功能需求、非功能需求、用例模型和用户故事，为系统设计、开发和测试提供明确的需求依据。

### 1.3 版本历史

| 版本 | 日期 | 作者 | 变更说明 |
|------|------|------|----------|
| 1.0 | 2026-04-24 | 苏鑫茹 | 初始版本 |
| 2.0 | 2026-04-27 | 苏鑫茹 | 新增信息采集功能需求 |

## 二、功能需求

### 2.1 功能需求总览

| 功能模块 | 功能点 | 需求来源 | 优先级 |
|----------|--------|----------|--------|
| 身份认证 | 手机号验证码登录 | 业务安全要求 | ⭐⭐⭐ |
| 信息采集 | 个人信息填写 | 报到业务需求 | ⭐⭐⭐ |
| 材料上传 | 身份证正反面照片上传 | 身份核验要求 | ⭐⭐⭐ |
| 数据管理 | 学生信息管理 | 后台管理需求 | ⭐⭐⭐ |

### 2.2 身份认证模块

#### 2.2.1 功能需求

| 需求编号 | 需求描述 | 说明 |
|----------|----------|------|
| FR-AUTH-001 | 支持手机号输入 | 用户输入11位中国大陆手机号 |
| FR-AUTH-002 | 验证码发送 | 点击按钮发送4位数字验证码 |
| FR-AUTH-003 | 发送频率限制 | 60秒内不可重复发送 |
| FR-AUTH-004 | 验证码校验 | 验证用户输入的验证码正确性 |
| FR-AUTH-005 | 登录状态保持 | 验证通过后建立Session |
| FR-AUTH-006 | 过期处理 | 验证码5分钟内有效 |

#### 2.2.2 业务规则

- 验证码格式：4位随机数字
- 有效期：5分钟
- 发送间隔：60秒
- 验证码使用后立即失效

### 2.3 信息采集模块

#### 2.3.1 功能需求

| 需求编号 | 需求描述 | 说明 |
|----------|----------|------|
| FR-INFO-001 | 姓名输入 | 学生真实姓名 |
| FR-INFO-002 | 学号输入 | 学生学号，唯一标识 |
| FR-INFO-003 | 手机号自动填充 | 从Session获取，不可编辑 |
| FR-INFO-004 | 表单验证 | 必填字段校验 |

#### 2.3.2 业务规则

- 学号必须唯一
- 手机号从登录Session自动获取
- 表单提交前必须完成验证

### 2.4 材料上传模块

#### 2.4.1 功能需求

| 需求编号 | 需求描述 | 说明 |
|----------|----------|------|
| FR-UPLOAD-001 | 身份证正面上传 | 人像面照片 |
| FR-UPLOAD-002 | 身份证反面上传 | 国徽面照片 |
| FR-UPLOAD-003 | 图片预览 | 选择后显示缩略图 |
| FR-UPLOAD-004 | 格式校验 | 仅支持.jpg/.png |
| FR-UPLOAD-005 | 大小限制 | 单张不超过5MB |
| FR-UPLOAD-006 | 重复检测 | 防止上传同一张照片 |
| FR-UPLOAD-007 | 正反面识别 | 自动识别并纠正顺序 |

#### 2.4.2 业务规则

- 必须同时上传正反两面
- 两张照片不能相同
- 系统自动识别正反面，顺序错误时自动纠正

### 2.5 数据管理模块

#### 2.5.1 功能需求

| 需求编号 | 需求描述 | 说明 |
|----------|----------|------|
| FR-MGMT-001 | 学生信息列表 | 查看所有报到学生 |
| FR-MGMT-002 | 身份证查看 | 点击链接查看原图 |
| FR-MGMT-003 | 验证码记录 | 查看验证码发送记录 |

## 八、接口需求

### 8.1 接口清单

| 接口名称 | 方法 | 路径 | 说明 |
|----------|------|------|------|
| 发送验证码 | POST | /api/send_code/ | 发送验证码到手机 |
| 验证码校验 | POST | /api/verify_code/ | 校验验证码并登录 |
| 信息提交 | POST | /api/register_with_info_v3/ | 提交报到信息和身份证照片 |

### 8.2 接口详细设计

#### 8.2.1 发送验证码接口

| 项目 | 内容 |
|------|------|
| 请求体 | `{"phone": "13800138000"}` |
| 成功响应 | `{"success": true, "message": "验证码已发送"}` |
| 失败响应 | `{"success": false, "message": "60秒内不可重复发送"}` |

#### 8.2.2 验证码校验接口

| 项目 | 内容 |
|------|------|
| 请求体 | `{"phone": "13800138000", "code": "1234"}` |
| 成功响应 | `{"success": true, "message": "登录成功"}` |
| 失败响应 | `{"success": false, "message": "验证码错误或已过期"}` |

#### 8.2.3 信息提交接口

| 项目 | 内容 |
|------|------|
| 请求体 | FormData（name、student_id、id_card_photo_front、id_card_photo_back） |
| 成功响应 | `{"success": true, "message": "报到成功"}` |
| 失败响应 | `{"success": false, "message": "请先登录"}` |

## 九、约束与限制

### 9.1 技术约束

- 开发阶段使用控制台打印验证码，生产环境需接入短信服务
- 图片存储在本地媒体目录，生产环境建议迁移至云存储
- 数据库使用SQLite，生产环境建议切换至PostgreSQL

### 9.2 业务约束

- 学号必须唯一，不允许重复报到
- 身份证照片必须同时上传正反两面
- 验证码有效期为5分钟，使用后立即失效

## 十、验收标准

### 10.1 功能验收

| 序号 | 验收项 | 预期结果 |
|------|--------|----------|
| 1 | 验证码发送 | 控制台打印4位数字验证码 |
| 2 | 频率限制 | 60秒内重复发送被拦截 |
| 3 | 验证码校验 | 正确验证码登录成功 |
| 4 | 错误处理 | 错误/过期验证码提示明确 |
| 5 | 手机号关联 | 自动填充且不可编辑 |
| 6 | 图片上传 | 预览正常，格式校验有效 |
| 7 | 正反面识别 | 顺序错误自动纠正 |
| 8 | 重复检测 | 同一张照片无法上传两次 |
| 9 | 数据存储 | 后台可查看完整信息和照片 |
| 10 | 权限控制 | 未登录无法访问提交接口 |

### 10.2 性能验收

| 序号 | 验收项 | 预期结果 |
|------|--------|----------|
| 1 | 验证码响应 | ≤1秒 |
| 2 | 图片上传 | ≤5秒 |
| 3 | 页面加载 | ≤3秒 |
